# max2tg

Мост между мессенджером [MAX](https://max.ru) и [Telegram](https://telegram.org). Бот подключается к MAX через WebSocket от имени вашего аккаунта и пересылает сообщения в указанные Telegram-чаты или топики в реальном времени.

## Возможности

- **Пересылка сообщений** - текст, фото, аудио, голосовые, видео, файлы
- **Пересланные сообщения** - отображаются с указанием исходного отправителя
- **Редактирование** - при изменении сообщения в MAX соответствующее сообщение в Telegram обновляется
- **Удаление** - удалённые сообщения либо удаляются из Telegram, либо помечаются маркером `[Удалено ...]` (настраивается)
- **Системные события** - вступление, добавление, удаление участников, создание чата
- **Синхронизация истории** - при запуске и после переподключения подгружаются последние сообщения, которые были пропущены
- **Маршрутизация** - каждый MAX-чат можно направить в отдельный Telegram-чат или топик
- **Автопереподключение** - при обрыве соединения бот переподключается с экспоненциальной задержкой
- **Debug-уведомления** - опционально присылает в Telegram личные сообщения об отключении и переподключении
- **Логирование** - каждый запуск пишет отдельный лог-файл

## Требования

- Go 1.21+
- Аккаунт в MAX
- Telegram-бот (токен от [@BotFather](https://t.me/BotFather))

## Установка и сборка

Вы можете загрузить готовую сборку для вашей ОС из раздела релизов.

Или собрать проект вручную:

```bash
git clone https://github.com/mochensky/max2tg
cd max2tg
go build -o max2tg .
```

С помощью PowerShell-скрипта (Windows):

```powershell
.\build.ps1
```

## Получение учётных данных MAX

Для работы бота нужны два значения из MAX: **токен** и **device ID**. Получить их можно через браузерные DevTools.

1. Откройте [web.max.ru](https://web.max.ru) и войдите в аккаунт.
2. Откройте DevTools (`F12` или `Ctrl+Shift+I`) - вкладка **Application** (Chrome/Edge) или **Storage** (Firefox).
3. Перейдите в **Local Storage** → `https://web.max.ru`.
4. Найдите ключ `__oneme_auth`.
4. Внутри найдите `token` - это ваш `MAX_TOKEN`.
5. Найдите ключ `__oneme_device_id` - это ваш `MAX_DEVICE_ID`.

> **Важно:** токен привязан к сессии. Если вы выйдете из аккаунта в браузере, токен станет недействительным и его нужно будет получить заново.

## Настройка

### 1. Первый запуск

```bash
./max2tg
```

При первом запуске программа создаёт два файла:

- `data/config.yml` - основная конфигурация
- `data/.env` - секретные переменные (токены)

После их создания программа завершится с просьбой заполнить данные.

### 2. Заполните `data/.env`

```env
MAX_TOKEN=ваш_токен_из_local_storage
MAX_DEVICE_ID=ваш_device_id_из_local_storage
TG_TOKEN=токен_вашего_telegram_бота
TG_DEBUG_USER_ID=ваш_telegram_user_id
```

`TG_DEBUG_USER_ID` - ваш числовой Telegram ID. Если указан, бот будет присылать вам личные сообщения при отключении и переподключении. Необязательное поле.

### 3. Настройте маршруты в `data/config.yml`

Основное, что нужно прописать - секция `chats`. Каждый маршрут связывает один MAX-чат с одним Telegram-чатом (или топиком).

```yaml
chats:
  # Маршрут без топика: MAX-чат -> Telegram-канал/группа
  - max_chat_id: 123456789
    telegram_chat_id: -1001234567890
    telegram_topic_id: 0

  # Маршрут с топиком: MAX-чат -> тема в Telegram-группе
  - max_chat_id: 987654321
    telegram_chat_id: -1001234567890
    telegram_topic_id: 5
```

**Как узнать `max_chat_id`:** откройте нужный чат в [web.max.ru](https://web.max.ru) - ID чата будет в URL.

**Как узнать `telegram_chat_id`:** добавьте бота в чат и воспользуйтесь [@getidsbot](https://t.me/getidsbot) или используйте сторонний Telegram клиент.

**Обратите внимание:** бот должен иметь права на отправку сообщений, на удаление сообщений, на отправку медиа и на управление топиками.

### 4. Остальные параметры `config.yml`

```yaml
# пути к данным
log_path: "data/logs"
db_path: "data/database.db"
download_path: "data/downloads"

# сколько последних сообщений в чате будет проверено на синхронизацию?
sync_history_depth: 30

# помечать ли удалённые сообщения маркером вместо удаления
save_deleted: true

# обрезать длинные сообщения вместо того, чтобы пропускать их (лимит заголовка: 1024 символа, лимит сообщения: 4096 символов)
truncate_long_messages: true

# параметры переподключения
max_retries: 5
base_retry_delay: 1s
ping_timeout: 90s
```

Секции `video_headers`, `audio_headers` и `user_agent` заполнены рабочими значениями по умолчанию и обычно не требуют изменений.

## Структура проекта

```
├── src/
│   ├── client.go          - высокоуровневый клиент MAX
│   ├── config.go          - загрузка и валидация конфига
│   ├── connection.go      - WebSocket-соединение и переподключение
│   ├── database.go        - SQLite: хранение ID сообщений
│   ├── enums.go           - константы и типы
│   ├── errors.go          - типы ошибок
│   ├── logger.go          - логирование
│   ├── models.go          - структуры данных
│   ├── parser.go          - парсинг ответов WebSocket API
│   ├── request_builder.go - построение запросов
│   ├── sender.go          - отправка в Telegram
│   ├── utils.go           - скачивание медиафайлов
│   └── version.go         - работа с версиями
└──  main.go               - точка входа, обработчики событий

```
